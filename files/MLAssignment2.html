Classification using a Support Vector Machine (SVM)¶

Now lets train a SVM model. This is done using scikit-learn, the default kernel is a Radial Basis Function (RBF). I decided to use the default RBF kernel based on the initial look of the data. The RBF is defined below:

$\phi(r) = e^{-(\gamma r)^2}$,

where $r = ||x-x'||$

$\gamma$ represents the influence a single training point will have. A large gamma means the Gaussian kernel has a large variance, and likewise, a large gamma implies a small variance.

Classification using K Nearest Neighbors (KNN)¶

Now I will be using a K Nearest Neighbors model. I am using the same generated data from above, so that the accuracy from each model can be compared. For this I am using the brute force algorithm. This algorithm finds the distance for all pairs of points in the dataset. This can be computationally expensive, but since our dataset has 100 points and 50 for training and testing, I picked this method. I have each neighbor weighed uniformly for my data. This is because the data is domain agnostic and semi-random based on the standard deviation I define when creating the data. If there was some intuition or predictability to the data then having the weights assigned by distance might be better. This would assign a weight based on the inverse of the distance to the neighbor, so the closer a point the more influence it has.

Results¶

Without much tuning of any hyperparameters, the KNN model was a little more accurate than the SVM model with k = 3. However, you can see from the last part that k plays a very important role. The accuracy was as low as 74% with k = 48. Although that is extreme since the training set only has 50 data points, it is useful to show the relationships here. This can also be seen by the random test point, when k = 48 it is not classified correctly. Also a k increases so does the computation time. Note that for not large increases in k, such as 2 and 4, the difference in time has a large variance. Sometimes k = 4 might run faster. This is based on me doing about 100 runs and the plot never looking consistent until largely increasing k.

To conclude, I think both models performed well with this dataset. The accuracy might be able to be improved with some hyperparameter tuning or different kernels for the SVM model. An interesting follow-on would be to see how these exact models perform on different data sets.
